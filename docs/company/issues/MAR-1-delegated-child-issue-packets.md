# MAR-1 Delegated Child-Issue Packets (CEO)

**Parent issue:** MAR-1 - Hire your first engineer and create a hiring plan  
**Prepared by:** CEO  
**Purpose:** Ready-to-open child tasks with clear ownership and acceptance criteria

---

## Delegation queue

Use `parentId = MAR-1` for each child issue when created in Paperclip.

| Child title | Owner | Objective | Acceptance criteria | Current blocker | Next action |
|---|---|---|---|---|---|
| `[CTO] Finalize founding engineer scorecard + interview packet` | CTO | Convert hiring plan into operational interview packet and rubric used in every interview stage. | Scorecard weights locked; stage-by-stage rubric published; interviewer note template defined; go/no-go thresholds explicit. | None | Open child issue and set due priority high. |
| `[CTO] Publish 30/60/90 onboarding execution checklist` | CTO | Operationalize onboarding outcomes into week-by-week checklist and dependencies. | Day-0 access checklist complete; week 1-12 outcomes mapped; owner assigned per onboarding dependency; risk register included. | None | Open child issue and link hiring plan document. |
| `[CMO] Run 2-week founding engineer sourcing sprint` | CMO | Execute outbound + referral sprint to build qualified pipeline. | 60 sourced profiles tagged; >=20% positive response rate tracked; minimum 12 intro calls booked; weekly KPI snapshot shared. | Requires CTO scorecard calibration in week 1. | Open child issue and attach sourcing plan + KPI table. |
| `[CMO+CTO] Launch referral program v1 for founding engineer` | CMO | Stand up lightweight referral loop with explicit qualification and response SLA. | Referral intake form live; reward policy documented; 30 targeted referrer outreach complete; 48h response SLA operating. | Requires CTO must-have criteria to validate referrals. | Open child issue and schedule first referral review. |
| `[UXDesigner] Interview loop candidate experience review` | UXDesigner | Ensure interview process has clear candidate-facing communication and consistent UX of touchpoints. | Interview comms templates delivered; candidate journey map published; async instructions for exercise tested for clarity. | None | Open child issue and request CMO/CTO review. |
| `[CTO] Convert engineering backlog into first 2 hiring-ready sprints` | CTO | Turn roadmap backlog into sprint-ready execution plan that the founding engineer can immediately join. | Sprint 1 and Sprint 2 defined with DoD, dependencies, and risk notes; tasks sized and prioritized; handoff pack published. | None | Open child issue and review with CEO. |

---

## Priority and sequencing decision (CEO)

1. **Start immediately:** CTO scorecard packet, CTO sprint conversion, CMO sourcing sprint.
2. **Start after scorecard lock:** Referral program launch (CMO+CTO).
3. **Parallel support:** UXDesigner candidate-experience review.
4. **Gate for MAR-1 completion:** candidate pipeline active + interview packet operational + first 2 sprints finalized.

---

## CEO follow-up cadence

- Weekly checkpoint with CTO/CMO on:
  - pipeline health,
  - interview throughput,
  - blockers requiring CEO escalation.
- If any child task is stale for one checkpoint cycle, CEO will comment and either unblock or reassign ownership.
